# Privacy Policy

Effective date: 2026-05-28

This Privacy Policy explains how `codex-english-buddy` handles data when installed and used as a Codex plugin.

Canonical repository: <https://github.com/laozeng1024/codex-english-buddy>

## Summary

`codex-english-buddy` is a local Codex plugin. It does not operate a hosted service, does not run centralized analytics, and does not send telemetry to the plugin author.

The plugin processes prompt text only to provide English coaching features such as translation, correction, refinement, reports, drills, and exports.

## Data Processed

Depending on configuration and user actions, the plugin may process:

- Prompt text submitted to Codex.
- Lightweight language or mode detection results, such as `translate`, `correct`, `refine`, or `clean`.
- Transformed prompt text when a verified full-history engine produces it.
- Correction annotations when a verified full-history engine produces them.
- Session identifiers provided by Codex, when available.
- Local plugin configuration.
- Local report and export files generated by the plugin.

## Default Engine

The default engine is `host_model`.

In `host_model` mode, the hook does not call a separate external model provider. The hook performs lightweight detection and injects instructions through Codex hook output so the active Codex model can show a translated, corrected, or refined prompt in the same conversation turn.

Because the hook does not receive the assistant's later transformed text, `host_model` history is limited. It records activity metadata and original prompt text, but it does not write guessed `corrected` text or guessed `annotations`.

No `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, or other provider credential is required for default behavior.

## Optional Full-History Engine

The optional `codex_cli` engine calls the user's local `codex exec` command in a child process. In this mode, prompt text is processed through the user's own Codex CLI session so the plugin can capture verified transformed text and annotations.

The child run is configured with recursion protection, hooks/plugins disabled for the child process, ephemeral execution, read-only sandboxing, and no approval prompts. If the child process fails, times out, or returns invalid structured output, the plugin falls back to `host_model` for that turn and writes a limited record.

`codex_cli` depends on the user's local Codex CLI installation, login state, configured model availability, and runtime environment.

## Future Provider Modes

A direct external provider API mode is not part of the first public version. If one is added later, it must be explicit opt-in and must state which provider receives prompt text.

## Local Storage

The plugin may store local configuration, history, and exports on the user's filesystem.

Typical local files include:

```text
.codex-english-buddy.json
~/.codex/codex-english-buddy/config.json
$CODEX_PLUGIN_DATA/history/YYYY-MM-DD.jsonl
$CODEX_PLUGIN_DATA/exports/english-buddy-export-*.md
$CODEX_PLUGIN_DATA/exports/english-buddy-export-*.csv
$CODEX_PLUGIN_DATA/exports/english-buddy-export-*.json
```

History powers reports such as `today`, `stats`, `mistakes`, `drill`, and `export`.

The `export` command writes only full-history records by default. Limited `host_model` records are counted in the export summary but their prompt text is not exported.

The `preview` command is a dry run and does not write normal prompt history.

## Sensitive Prompts

Users can configure `sensitive_patterns` to skip prompts that match project-specific secrets or private text. Skipped prompts do not receive English Buddy context and do not create history records.

Users should avoid submitting secrets, credentials, private keys, access tokens, or highly sensitive personal information to Codex or to this plugin.

## Data Sharing

The plugin author does not receive prompt text, configuration, history, exports, telemetry, or analytics from the plugin.

Data may be processed by Codex itself according to the user's Codex environment and configuration. When `codex_cli` is enabled, prompt text is processed through the user's own local Codex CLI session.

## Data Retention

The plugin keeps local data until the user deletes it. The plugin does not provide centralized storage and does not retain data on behalf of users.

## Data Deletion

To remove local plugin data, delete:

```text
~/.codex/codex-english-buddy/
$CODEX_PLUGIN_DATA/
.codex-english-buddy.json
```

Exact paths can vary by Codex host and installation method. The `doctor` command can report the active data path for the current environment.

## Security

The plugin is designed to run locally and to avoid direct provider credentials in the default mode. Users are responsible for protecting their local filesystem, Codex account, local configuration files, and exported reports.

Exported files may contain original prompt text and transformed text from full-history records. Treat exported files as user data and review them before sharing or committing them.

## Changes

This policy may be updated as the plugin evolves. Material behavior changes, especially changes involving provider integrations or data transmission, should be documented before release.
